# 使用手册

## 适用场景

这个扩展适合把知乎内容导出到本地继续整理，比如：

- 归档自己写过的文章和回答
- 把知乎内容迁移到 Obsidian、Typora、Notion 或博客
- 保存问题页多个回答用于横向比较
- 把首页推荐、关注流和热榜整理成选题清单

## 支持的页面

| 页面 | 识别方式 | 推荐导出方式 | 结果类型 |
| --- | --- | --- | --- |
| 专栏文章 | `/p/...` | 弹窗或悬浮球 | 单篇 Markdown |
| 单条回答 | `/question/.../answer/...` | 弹窗或悬浮球 | 单篇 Markdown |
| 问题页 | `/question/...` | 悬浮球 | 多回答整合 Markdown |
| 首页推荐 | `/` | 悬浮球 | 信息流列表 Markdown |
| 关注动态 | `/follow` | 悬浮球 | 信息流列表 Markdown |
| 热榜 | `/hot` | 悬浮球 | 热榜清单 Markdown |

## 基本使用流程

### 方式一：通过弹窗导出

1. 打开支持的知乎页面。
2. 点击浏览器工具栏中的扩展图标。
3. 等待弹窗识别页面类型、标题和作者。
4. 点击 `导出 Markdown`。

单篇文章和单条回答最适合直接使用弹窗导出。

### 方式二：通过悬浮球导出

1. 打开支持的知乎页面。
2. 确认右下角出现悬浮导出按钮。
3. 点击悬浮球触发导出。
4. 复杂页面会自动滚动、加载内容并切换到对应导出器。

问题页、首页推荐、关注动态和热榜页建议优先使用悬浮球。

## 设置项说明

### 悬浮导出按钮

- 开启后，在支持页面右下角显示悬浮球
- 支持拖拽位置、靠边停靠和位置记忆
- 关闭后会立即移除当前页面上的悬浮球

### 最大下载回答数

- 范围为 `1-50`
- 默认值是 `20`
- 问题页批量导出和信息流导出都会使用这个上限
- 如果页面实际加载条数不足，导出结果会少于设置值

### 下载图片到本地

- 只影响单篇文章和单条回答导出
- 开启后不再输出单独 `.md` 文件，而是输出 ZIP
- ZIP 中包含：
  - 一个 Markdown 文件
  - 一个 `images/` 文件夹
- Markdown 中图片路径会变为 `images/xxx.jpg`

## 每种页面的导出结果

### 专栏文章 / 单条回答

输出内容包括：

- YAML Front Matter
- 标题、作者、原始链接
- 导出日期
- 文章创建时间和编辑时间
- 转换后的正文 Markdown

### 问题页

输出内容包括：

- 问题标题
- 导出日期
- `answer_count`
- 每条回答的作者、时间和正文

问题页会先滚动加载更多回答，再按顺序整理。

### 首页推荐 / 关注动态

输出内容包括：

- 页面类型标题
- 导出日期
- `item_count`
- 每条内容的标题、作者、时间、原文链接和正文摘录

导出器会尽量点击“阅读全文”并对重复标题做去重。

### 热榜

输出内容包括：

- 热榜标题
- 导出日期
- `count`
- 每条热榜的排名、标题、链接、热度

## Markdown 转换规则

扩展内置了一套面向知乎页面的 Turndown 规则，主要包括：

- 公式：将知乎公式节点转换为 LaTeX
- 图片：优先取高分辨率地址
- 代码块：转换为 fenced code block
- 链接卡片：转换为普通链接
- 表格：转换为 Markdown 表格
- 视频卡片：保留为可点击链接
- 换行、分隔线、常见格式标签：按 Markdown 语义输出

## 结果示例

### 单篇内容 Front Matter

```yaml
---
title: "示例标题"
url: https://www.zhihu.com/question/...
author: 示例作者
date: 2026-03-31
created: 2025-12-01
edited: 2025-12-03
---
```

### 多回答导出头部

```yaml
---
title: "示例问题"
url: https://www.zhihu.com/question/...
date: 2026-03-31
answer_count: 12
---
```

## 已知限制

- 只能导出当前页面已加载的内容
- 默认不会抓取评论区
- 被折叠、未展开或受权限限制的内容，只有在页面中实际可见时才可能被导出
- 知乎改版后，选择器可能需要同步更新
- 图片本地下载模式不会用于问题页、信息流和热榜的批量导出
